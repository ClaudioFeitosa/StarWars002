## REPAIR Change-Log Format

When MODE == REPAIR, for each modified row, log:

```
CHANGE: {item_id} | field: {field_name} | old: "{old_value}" | new: "{new_value}" | reason: {directive reason}
PRESERVED: {item_id} | all fields unchanged (no directive matched)
```

Accumulate these in CHANGE_LOG throughout Step 4.
Write the full CHANGE_LOG in the REPAIR Changes section above.

If no changes were made (feedback pointed to a non-existent item):
Write: `No matching items found for REPAIR directive. Existing CSV preserved verbatim.`

## Version Increment Rules

| Mode | Version Increment | Example |
|------|------------------|---------|
| BUILD | Start at 1.0.0 | 1.0.0 |
| REPAIR (field corrections) | Patch increment | 1.0.0 → 1.0.1 |
| REPAIR (scope changes) | Minor increment | 1.0.0 → 1.1.0 |
| REPAIR (full regeneration) | Minor increment | 1.0.0 → 1.1.0 |
